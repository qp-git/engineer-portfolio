# Publication Guidelines

このリポジトリは公開用ポートフォリオとして運用するため、掲載する情報を以下の基準で整理します。

## 公開してよい情報

- プロジェクト概要
- 使用技術
- 担当範囲
- 構成概要
- 設計判断
- 苦労した点
- 学んだこと
- 公開可能なサンプルコード
- 公開可能な構成図

## 公開しない情報

- 実名入りスキルシート
- メールアドレス、電話番号などの個人情報
- 顧客名、企業名、社内サービス名など、公開許可がない固有名詞
- APIキー、秘密鍵、パスワード、トークン
- 面接練習ログ、採点結果、弱点管理
- 非公開案件の詳細な構成情報

## 方針

公開リポジトリは「成果物・設計意図・学び」を示す場所とし、面接練習や自己評価のための内部メモは private リポジトリまたはローカルで管理します。
