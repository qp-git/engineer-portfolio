# Discordお題Bot / OdaiBox

## 概要

OdaiBoxは、Discord上でゲーム用のお題をランダムに出すBotです。

Discordで `/odai` を実行すると、次の試合で使うお題をランダムに返します。単にランダム抽選するだけでなく、非エンジニアでもブラウザからお題を追加・編集・ON/OFFできる管理画面を用意しました。

## 作成背景

Discordで通話しながらゲームをするとき、同じ遊び方が続くとマンネリ化しやすい課題がありました。

そこで、次の試合にちょっとした制約やお題を出し、遊び方に変化をつけるBotを作成しました。

ただし、お題をLambdaのコード内に固定してしまうと、変更のたびに開発者がコードを修正してデプロイする必要があります。そのため、利用者側でサーバーの雰囲気に合わせてカスタムできる管理UIを重視しました。

## 実装内容

- `/odai` コマンドでランダムなお題を返すBotを作成
- DynamoDBでサーバーごとのお題を管理
- Web管理画面からお題の追加・編集・ON/OFFを可能にした
- 管理画面をS3 + CloudFrontで静的配信
- API Gateway + Lambda + DynamoDBによるサーバーレス構成を採用
- `/admin-login` による一時パスワード方式を検討・実装
- 保存する情報と保存しない情報を整理し、プライバシーポリシーを作成

## 工夫した点

一番重視したのは、Botを作った本人だけが管理するのではなく、ITに詳しくない友達でも自分たちのサーバー用に育てられるようにした点です。

Bot利用者はDiscordで `/odai` を実行するだけで使えます。お題を編集したい場合は、Web管理画面からタイトル、説明文、ON/OFFを変更できるようにしました。

また、常時起動するサーバーを持たず、必要なときだけLambdaを実行する構成にすることで、個人開発でも継続しやすい低コスト構成を意識しました。

## セキュリティ・プライバシー面で考えたこと

管理画面を用意すると、URLを知っているだけで誰でも編集できてしまうリスクがあります。

そのため、固定ID・固定パスワードではなく、Discord上の `/admin-login` コマンドを入口にして、権限のあるユーザーに一時パスワードを発行する方式を考えました。

また、BotはDiscordのサーバーIDやユーザーIDを扱います。これらはパスワードのような秘密情報ではありませんが、どのサーバーで誰が使ったかの手がかりになり得るため、機能に必要な範囲だけ保存する方針にしました。

保存しない情報として、音声通話、チャット本文、DM、Discordのパスワードを明示しました。

## 学び

このプロジェクトで学んだのは、個人開発でも「動くものを作る」だけでは不十分だという点です。

使う人が非エンジニアであれば、機能そのものだけでなく、管理画面の分かりやすさ、変更しやすさ、保存する情報の説明まで含めて設計する必要があります。

また、サーバーレス構成は低コストで個人開発に向いていますが、管理画面からAPIを操作できるようにすると、認証・認可・プライバシー説明が重要になると学びました。
