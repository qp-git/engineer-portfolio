# Discordお題Bot / OdaiBox

## 概要

OdaiBoxは、Discordで通話しながらゲームをするコミュニティ向けに、お題をランダムに出すBotです。

Discordで `/odai` を実行すると、次の試合で使うお題をランダムに返します。

単にランダム抽選するだけでなく、非エンジニアでもブラウザからお題を追加・編集・ON/OFFできるWeb管理画面を用意した点が特徴です。

## 作成背景

Discordで通話しながらゲームをするとき、いつも同じ遊び方が続くとマンネリ化しやすい課題がありました。

コミュニティ内で「今回はこういう遊び方をしてみよう」と決めることはありますが、それを毎回人が考えるのではなく、Botによってランダムにお題として出せるようにすることを目的としました。

また、お題をLambdaのコード内に固定してしまうと、変更のたびに開発者がコードを修正してデプロイする必要があります。

そのため、利用者側でDiscordサーバー（コミュニティ）の雰囲気に合わせて、お題を追加・編集できるWeb管理画面を重視しました。

## 使用技術

### Discord側

| 技術 | 役割 |
|---|---|
| Discord Slash Commands | `/odai` や `/admin-login` の実行入口 |
| Discord Interaction | コマンド実行時のリクエスト送信 |

### AWS側

| サービス | 役割 |
|---|---|
| API Gateway | Discordや管理画面からのリクエスト入口 |
| Lambda | お題抽選、管理画面ログイン、保存・編集処理 |
| DynamoDB | お題データ、設定、一時ログイン情報の保存 |
| S3 | Web管理画面の静的ファイル配置 |
| CloudFront | Web管理画面の配信 |

### 管理画面

| 技術 | 役割 |
|---|---|
| HTML / CSS / JavaScript | お題の追加・編集・ON/OFF切り替え |

## 実装内容

- `/odai` コマンドでランダムなお題を返すBotを作成
- DynamoDBでDiscordサーバー（コミュニティ）ごとのお題を管理
- Web管理画面からお題の追加・編集・ON/OFFを可能にした
- 管理画面をS3 + CloudFrontで静的配信
- API Gateway + Lambda + DynamoDBでお題抽選・保存処理を実装
- `/admin-login` による一時パスワード方式を検討・実装
- 保存する情報と保存しない情報を整理し、プライバシーポリシーを作成

## 工夫した点

一番重視したのは、Botを作った本人だけが管理するのではなく、ITに詳しくない友達でも自分たちのコミュニティ用に育てられるようにした点です。

Bot利用者はDiscordで `/odai` を実行するだけで使えます。お題を編集したい場合は、Web管理画面からタイトル、説明文、ON/OFFを変更できるようにしました。

これにより、開発者が毎回コードを修正しなくても、利用者側でコミュニティの雰囲気に合わせてお題を調整できます。

## セキュリティ・プライバシー面で考えたこと

管理画面を用意すると、URLを知っているだけで誰でも編集できてしまうリスクがあります。

そのため、固定ID・固定パスワードではなく、Discord上の `/admin-login` コマンドを入口にして、権限のあるユーザーに一時パスワードを発行する方式を考えました。

また、BotはDiscordサーバーIDやユーザーIDを扱います。これらはパスワードのような秘密情報ではありませんが、どのコミュニティで誰が使ったかの手がかりになり得るため、機能に必要な範囲だけ保存する方針にしました。

保存しない情報として、音声通話、チャット本文、DM、Discordのパスワードを明示しました。

## 学び

このプロジェクトで特に学んだことは、Bot本体だけでなく、利用者が操作するWeb管理画面を用意すると、認証・認可・プライバシー説明も設計対象になるという点です。

お題をコード内に固定するだけであれば構成は単純です。しかし、利用者がブラウザからお題を追加・編集できるようにすると、誰が管理画面を操作できるのか、どのDiscordサーバー（コミュニティ）のお題を編集できるのか、どの情報を保存するのかを明確にする必要があります。

そのため、`/admin-login` による一時ログイン方式や、保存する情報・保存しない情報を整理したプライバシーポリシーを用意しました。
