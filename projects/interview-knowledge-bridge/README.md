# Interview Knowledge Bridge

## 概要

Interview Knowledge Bridge は、Custom GPT から GitHub 上の許可済み Markdown を取得できるようにするための中継 API です。

公開ポートフォリオは人間が読みやすいように整理し、細かな躓き、判断理由、公開資料には載せにくい細かな躓きや判断理由は、AI に渡す補助コンテキストとして分けて管理します。

この API では、GitHub 上のすべてのファイルを AI に参照させるのではなく、ポートフォリオ整理や回答生成に使ってよい Markdown だけをホワイトリストで管理します。これにより、Custom GPT が参照する情報の範囲を明確にし、関係ないファイルや古いメモが混ざることを防ぎます。

## 作成背景

ポートフォリオを整理する中で、人間に見せる公開資料と、AI に参照させたい補足情報では、適した情報量が違うと感じました。

公開資料では、初見の人が読みやすいように、概要、構成、設計判断、学びを簡潔にまとめる必要があります。一方で、ポートフォリオ整理では、実際に詰まった点、なぜその構成にしたか、他の選択肢をどう考えたか、今ならどう改善するかといった細かい情報も役立ちます。

そこで、公開ポートフォリオとは別に、AI に渡す補助コンテキストを Markdown として管理し、Custom GPT から必要な文書だけ取得できる中継 API を作成しました。

## 目的

このプロジェクトの目的は、Custom GPT に対して、参照してよい情報だけを明示的に渡すことです。

主な目的は以下です。

- 人間向けの公開ポートフォリオと、AI に渡す補助コンテキストを分ける
- Custom GPT が参照できる Markdown をホワイトリストで制御する
- AI の回答生成に使ってよい情報だけを API 経由で返す
- 関係ないファイルや古いメモを AI に読ませない
- 公開資料と補足メモが増えても、参照範囲を同じ仕組みで管理できるようにする

## 構成の概要

基本的な流れは以下です。

    Custom GPT
      ↓ Actions 経由でリクエスト
    API Gateway
      ↓
    Lambda
      ↓ 許可済みドキュメントか確認
    Whitelist
      ↓ 対象 Markdown を取得
    GitHub API
      ↓ Markdown を返却
    Custom GPT

Bridge API は、Custom GPT からのリクエストを受け取り、ホワイトリストに登録された Markdown だけを取得して返します。

## 使用技術・要素

- Custom GPT Actions
- OpenAPI schema
- Amazon API Gateway
- AWS Lambda
- GitHub API
- GitHub 上の Markdown
- ホワイトリストによる参照制御

## 実装・検証したこと

- Custom GPT から外部 API を呼び出す構成を検証
- API Gateway と Lambda を使った中継 API を作成
- GitHub API 経由で Markdown を取得する流れを整理
- 参照可能な Markdown をホワイトリストで制御
- 人間向けの公開ポートフォリオと、AI が読む補足情報を分けて管理
- ポートフォリオ整理で必要な補足情報を、必要なときだけ取得できる構成を検討

## このプロジェクトで学んだこと

このプロジェクトでは、AI に情報を渡す場合、単にデータを置いておくだけでなく、どの情報を読ませるかを設計する必要があると学びました。

AI 補助用の AI では、公開ポートフォリオのように読みやすく整理された情報だけでなく、細かな躓きや判断理由、補足説明も回答の材料になります。一方で、すべての情報を読ませると、関係ない情報や古いメモが混ざり、回答の精度が下がる可能性があります。

そのため、AI が参照する文書をホワイトリストで管理し、必要な情報だけを API 経由で渡す構成にしました。

この経験から、AI 活用では、モデルそのものだけでなく、参照させる情報の粒度、範囲、管理方法も重要な設計対象になると学びました。
