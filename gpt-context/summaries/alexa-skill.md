# Alexa Skill

## 概要

Alexaスキルを題材に、音声UIとAWS Lambdaを連携させた朝の情報取得プロジェクト。

自宅と勤務地の2地点の天気を取得し、Alexaに読み上げさせる仕組みを作成した。

当初は電車の遅延情報も読み上げる想定だったが、個人開発で利用できる運行情報APIの制約により、実証は天気取得を中心に整理している。

## 使用技術

- Alexa Skills Kit
- AWS Lambda
- Node.js
- Open-Meteo
- 外部API連携
- CloudWatch Logs
- Lambda環境変数

## 構成

    Alexa
      ↓ カスタムスキル
    AWS Lambda
      ├─ 天気API
      └─ 遅延情報API 調査
      ↓
    読み上げ文を生成
      ↓
    Alexaが読み上げ

## 実装内容

- Alexaカスタムスキルを作成
- LambdaをAlexa Skills Kitのエンドポイントとして設定
- Lambdaから外部APIを呼び出し、2地点の天気を取得
- 取得結果を読み上げ文に整形
- Lambda環境変数で地点情報を管理
- CloudWatch Logsで実行確認
- 開発中はInvocation Name、実運用は定型アクションで起動する方針を整理

## 学び

- Alexa、Lambda、外部APIの役割分担を理解した
- 音声UIでは起動しやすさ、聞き取りやすさ、情報量の調整が重要
- 外部API連携では、コードだけでなく利用条件や契約条件の確認が必要
- 遅延情報APIでは、実装ではなく個人利用可能なデータ提供条件がボトルネックになった
- 技術検証では、成功しなかった理由を切り分けて説明できることにも価値がある

## 特に強調したい学び

このプロジェクトで特に大きかった学びは、Alexa Skill が Lambda の「入口」と「出口」の両方を担えるという点です。

Alexa は、ユーザーの音声入力を受け取り、インテントとして Lambda を起動するトリガーになります。さらに、Lambda の処理結果を Alexa に返すことで、ユーザーへ音声で応答できます。

つまり、Alexa Skill では、音声入力をきっかけに Lambda を実行し、その結果を音声で返す一連の流れを作れます。

当初は、外部から Lambda を呼び出すには API Gateway が必要だと考えていました。しかし Alexa Skill では、Alexa Skills Kit のエンドポイントとして Lambda 関数の ARN を指定できるため、API Gateway を挟まずに構成できます。

この経験から、Alexa は単なる音声入力の窓口ではなく、Lambda を起動する音声トリガーであり、処理結果を返す音声インターフェースでもあると理解しました。

もう一つの学びは、外部APIは技術的に呼び出せるかだけでなく、利用条件や提供範囲まで確認する必要があるという点です。

天気APIは比較的検証しやすかった一方で、遅延情報APIは無料プランや個人利用では扱える情報に制限があり、法人契約や有料契約が必要になる場合があると分かりました。

そのため、このプロジェクトでは、Alexa Skill と Lambda の連携だけでなく、外部APIの利用条件を確認した上で実装可否を判断することも学習成果として整理しています。
